### 1.2 存在的问题 | Identified Issues

#### 问题1: 架构表述过时 | Outdated Architecture Description

**当前问题:**
- 首页提到"4维度分析矩阵",但实际系统已升级为**5层架构**
- 没有展示 Layer 5→4→3→2→1 的完整处理流程
- 缺少对**子步骤系统**的说明(如Layer 5有5个子步骤)

**影响:**
- 用户无法理解系统的真实工作原理
- 与实际使用流程不符,降低信任度

#### 问题2: 核心技术亮点展示不足 | Insufficient Technical Highlights

**当前缺失:**
- ✘ CAASS v2.0 评分系统
- ✘ 18点 De-AIGC 改写技术
- ✘ 词汇锁定系统 (Step 1.0)
- ✘ 句式多样性分析
- ✘ 锚点密度检测
- ✘ 语义回声生成
- ✘ PPL困惑度可视化 (🤖/⚠️/🤯)
- ✘ 结构可预测性评分

**影响:**
- 无法体现技术深度和独特性
- 竞争力不明显

#### 问题3: 用户旅程不清晰 | Unclear User Journey

**当前问题:**
- "How It Works" 仅展示4个高层步骤
- 没有说明5层处理的顺序和必要性
- 缺少流程可视化图

**理想用户旅程:**
```
上传文档
  ↓
Step 1.0: 词汇锁定 (用户确认专业术语)
  ↓
Layer 5: 文档层分析 (5个子步骤)
  ↓
Layer 4: 章节层分析 (5个子步骤)
  ↓
Layer 3: 段落层分析 (5个子步骤)
  ↓
Layer 2: 句子层分析 (句式、合并、拆分)
  ↓
Layer 1: 词汇层改写 (AIGC指纹消除)
  ↓
导出优化后的文档
```

#### 问题4: 价值主张不够鲜明 | Weak Value Proposition

**当前表述:**
- "人机协作式AIGC文本优化" - 过于宽泛
- "直击Turnitin & GPTZero核心算法" - 需要更具体的证明

**需要强化:**
1. **唯一性** - 市面上唯一支持5层颗粒度分析的工具
2. **专业性** - 基于学术论文特性设计的专业系统
3. **可控性** - 用户完全掌控每一层的修改决策
4. **可解释性** - 每个问题都有明确的检测逻辑和修改理由

#### 问题5: 缺少社会证明 | Missing Social Proof

**当前缺失:**
- ✘ 没有使用案例
- ✘ 没有效果数据 (如"平均降低AI检测率80%")
- ✘ 没有技术对比表

## 二、UX设计原则 | UX Design Principles

### 2.1 核心设计目标 | Core Design Goals

| 目标 Goal | 说明 Description |
|-----------|------------------|
| **建立信任** | 展示技术深度和专业性,消除"黑盒工具"的疑虑 |
| **Build Trust** | Show technical depth and professionalism |
| **降低理解门槛** | 复杂技术简单化,用可视化图表辅助理解 |
| **Lower Barrier** | Simplify complex tech with visual aids |

| **Highlight Uniqueness** | Emphasize 5-layer architecture advantages |

| **Drive Action** | Clear CTAs to encourage immediate trial |

### 2.2 目标用户画像 | Target User Personas

#### Persona 1: 学术研究者 | Academic Researcher

**特征:**
- ESL(非英语母语)背景
- 需要发表英文论文
- 担心AI检测被拒稿
- 重视学术严谨性

**关注点:**
- ✓ 是否能保留专业术语?
- ✓ 语义会不会被改变?
- ✓ 能否通过Turnitin检测?
- ✓ 修改建议是否可靠?

**设计策略:**
- 强调"术语锁定"功能
- 展示"语义相似度验证"
- 提供检测器视角对比
- 突出"人机协作"而非"自动替换"

#### Persona 2: 效率优先用户 | Efficiency-focused User

**特征:**
- 时间紧迫
- 希望快速处理
- 对技术细节不敏感
- 相信工具效果

**关注点:**
- ✓ 多久能处理完?
- ✓ 是否需要逐句确认?
- ✓ 能否自动处理?

**设计策略:**
- 突出"YOLO模式"
- 展示处理速度
- 简化操作流程说明

#### Persona 3: 技术好奇者 | Tech-savvy Explorer

**特征:**
- 了解NLP技术
- 想知道"怎么做到的"
- 对算法感兴趣
- 可能是竞品分析

**关注点:**
- ✓ 使用什么算法?
- ✓ PPL怎么计算?
- ✓ 为什么是5层架构?
- ✓ 和其他工具有何区别?

**设计策略:**
- 展示技术架构图
- 列出核心算法
- 提供技术白皮书链接
- 展示技术对比表

### 2.3 信息架构设计 | Information Architecture

#### 金字塔原则 | Pyramid Principle

```
                  核心价值主张
                (3秒内抓住注意力)
                       ▲
                      / \
                     /   \
                    /     \
                   /       \
          技术亮点           使用场景
        (30秒了解优势)    (1分钟判断是否适用)
                   \       /
                    \     /
                     \   /
                      \ /
                       ▼
                  详细功能说明
              (3-5分钟深入了解)
```

**内容层级:**

1. **Level 1 (3秒)**: Hero Section
   - Slogan: "5层架构,从骨到皮的De-AIGC引擎"
   - Sub-slogan: "市面唯一支持文档→章节→段落→句子→词汇全颗粒度分析的学术论文优化工具"

2. **Level 2 (30秒)**: 核心差异化特性
   - 5层架构可视化
   - 3大核心技术
   - 双模式/双轨道

3. **Level 3 (1分钟)**: 适用场景
   - 谁需要?
   - 解决什么问题?
   - 不适合什么场景?

4. **Level 4 (3分钟)**: 详细功能
   - 技术细节
   - 工作流程
   - 效果展示

5. **Level 5 (5分钟)**: 深度内容
   - 技术白皮书
   - 算法说明
   - FAQ

**Why 5-Layer?**

```
为什么需要5层架构?

问题: 如果先改句子(Layer 2)再调结构(Layer 5),结构调整可能导致句子级修改失效

解决方案: 必须按 Layer 5 → 4 → 3 → 2 → 1 顺序处理

类比: 就像装修房子
- Layer 5: 先看整体布局(房间分布合理吗?)
- Layer 4: 再看各房间功能(卧室、客厅搭配合理吗?)
- Layer 3: 然后看墙面设计(墙壁、窗户协调吗?)
- Layer 2: 接着调整家具摆放(沙发、桌子位置合理吗?)
- Layer 1: 最后装饰细节(摆件、挂画等)

如果直接从装饰细节开始,可能摆好了又因为改布局而推倒重来。
```

#### Section 6: Detection Logic Explained

**Turnitin 检测原理:**
```
Turnitin的AI检测逻辑:
1. 训练数据分类器 - 基于大量AI生成样本训练
2. 全文风格分析 - 检测整体写作风格一致性
3. 段落结构模式 - 识别AI常见的段落组织模式
4. 引用模式检测 - 分析引用的自然性

AcademicGuard应对策略:
✓ Layer 5结构重组 - 打破AI固有的段落模式
✓ Layer 4章节优化 - 调整章节过渡自然性
✓ Layer 3段落多样性 - 增加段落结构变化
```

**GPTZero 检测原理:**
```
GPTZero的核心指标:
1. Perplexity (困惑度) - 文本的不可预测性
   AI文本: PPL < 20 (过于平滑)
   人类文本: PPL > 40 (充满意外)

2. Burstiness (突发性) - 句子长度和结构的变化度
   AI文本: 句长变异系数 < 0.25 (过于均匀)
   人类文本: 句长变异系数 > 0.35 (富有节奏)

AcademicGuard应对策略:
✓ Layer 2句子合并拆分 - 增加句长变异
✓ Layer 1词汇多样化 - 提升困惑度
✓ 18点改写技术 - 全面优化PPL和Burstiness
```

#### Section 7: Dual Mode Comparison

**对比表:**

| 特性 | 干预模式 Intervention | YOLO模式 Auto |
|------|----------------------|---------------|
| **适用场景** | 重要论文、高质量要求 | 时间紧迫、快速处理 |
| **操作方式** | 逐层手动选择方案 | 全自动处理L5→L1 |
| **处理时间** | 较长(需人工确认) | 较短(自动应用) |

#### Section 10: FAQ & Resources

**常见问题:**

Q1: 为什么必须先做词汇锁定(Step 1.0)?
A1: 锁定的专业术语会自动传递到所有后续LLM调用的Prompt中,确保改写时不会误改专业词汇。如果先改写再锁定,已经改过的内容可能包含错误替换。

Q2: 5层架构和传统工具有何区别?
A2: 传统工具只在句子级做同义词替换,无法改变文章的结构特征。5层架构从文档→词汇逐层优化,能从根本上改变AI文本的特征模式。

Q3: 使用YOLO模式安全吗?
A3: YOLO模式会自动选择最优建议,但仍建议最后人工审核。重要论文建议使用干预模式,逐层确认修改。

Q4: 支持中文论文吗?
A4: 目前仅支持英文学术论文。中文论文的AIGC检测逻辑不同,暂不支持。

Q5: 价格如何计算?
